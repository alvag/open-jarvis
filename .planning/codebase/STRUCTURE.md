## Directory Purposes

**src/:**
- Purpose: All TypeScript source code
- Contains: Layered application logic
- Key patterns: ES modules, strict typing, no relative imports (file extensions required)

**src/agent/:**
- Purpose: Agent loop and context building
- Contains: Agent orchestration logic and system prompt synthesis
- Key files: `agent.ts` (main loop), `context-builder.ts` (soul + memories + date)

**src/llm/:**
- Purpose: Language model abstraction and routing
- Contains: Provider interface, OpenRouter client, model selection logic
- Key patterns: Complexity-based model selection (simple/moderate/complex → different models)

**src/memory/:**
- Purpose: Persistent state management
- Contains: SQLite wrapper, memory CRUD, session management, full-text search
- Key patterns: Factory pattern for MemoryManager, migrations versioned in db.ts

**src/tools/:**
- Purpose: Agent capabilities registry
- Contains: Tool interface, registry, and all tool implementations
- Key patterns: Each tool exported as `{definition, execute}` object

**src/channels/:**
- Purpose: Platform adapters
- Contains: Channel interface, Telegram implementation
- Key patterns: Message handler callback pattern, attachment handling, typing indicator

**data/:**
- Purpose: Runtime data directory
- Contains: SQLite database (WAL mode for crash safety), uploaded files
- Generated: Yes (created at runtime)
- Committed: No (except as placeholder)

**dist/:**
- Purpose: Compiled JavaScript output
- Generated: Yes (by `tsc`)
- Committed: No

## Key File Locations

**Entry Points:**
- `src/index.ts`: Main bot - initializes all systems, starts message loop
- `src/supervisor.ts`: Process supervisor - spawns and manages bot with restarts

**Configuration:**
- `src/config.ts`: Typed environment variable loader
- `.env`: Secrets file (not committed)
- `soul.md`: Agent personality and instructions (committed, must exist)
- `AGENTS.md`: Optional agent operating rules (loaded if present)

**Core Logic:**
- `src/agent/agent.ts`: Agent loop - receives message, calls LLM, executes tools, returns response
- `src/agent/context-builder.ts`: System prompt builder - combines soul + memories + context
- `src/llm/openrouter.ts`: LLM client - translates agent format to OpenAI-compatible API

**Database:**
- `src/memory/db.ts`: Schema and migration system
- `src/memory/memory-manager.ts`: Query builders and API methods
- `data/jarvis.db`: Actual SQLite database file

**Tool System:**
- `src/tools/tool-registry.ts`: Registry - maps names to executors
- `src/tools/tool-types.ts`: TypeScript interfaces for all tools
- `src/tools/built-in/`: All tool implementations

**Telegram Integration:**
- `src/channels/telegram.ts`: Message handling, photo/document download, typing indicator, rate limiting

## Naming Conventions

**Files:**
- `kebab-case.ts` for files: `get-current-time.ts`, `tool-registry.ts`, `save-memory.ts`
- Exception: `index.ts`, `AGENTS.md`, `soul.md` (conventions or entry points)

**Directories:**
- `kebab-case/` for folders: `built-in/`, no `index.ts` barrel exports (direct imports)

**Functions & Variables:**
- `camelCase` for functions and variables: `runAgent()`, `buildSystemPrompt()`, `createMemoryManager()`
- `SCREAMING_SNAKE_CASE` for constants: `MAX_ITERATIONS`, `UPLOADS_DIR`, `STABLE_THRESHOLD`

**Types & Interfaces:**
- `PascalCase` for types: `ChatMessage`, `AgentContext`, `Tool`, `MemoryManager`
- Prefix "I" not used (TypeScript convention avoidance)

**Exports:**
- Default export for tools: `export default getCurrentTime;`
- Named exports for utilities: `export function runAgent() {}`
- Avoid barrel files (no `src/tools/index.ts` re-exporting all)

## Where to Add New Code

**New Tool:**
- Create file: `src/tools/built-in/your-tool-name.ts`
- Export tool object: `const yourTool: Tool = { definition: {...}, async execute(...) {...} }`
- Import and register in `src/index.ts`: `toolRegistry.register(yourTool)`
- Add conditional enable flag in `config.ts` if external service

**New Channel:**
- Create file: `src/channels/your-channel.ts` implementing `Channel` interface
- Methods: `start(handler: MessageHandler): Promise<void>`, `stop(): Promise<void>`
- IncomingMessage properties: userId, userName, text, channelId, rawEvent, attachments (optional)
- ChannelResponse: text string, images array (optional)
- Instantiate in `src/index.ts`: `const yourChannel = new YourChannel(...)`
- Call: `await yourChannel.start(async (msg) => { return runAgent(...); })`

**New Command (direct, bypassing agent):**
- Location: `src/channels/telegram.ts` in `start()` method
- Pattern: `this.bot.command("name", async (ctx) => { /* handle directly */ })`
- Example: `/restart` command exits with code 101 for supervisor to restart

**New Memory Feature:**
- Add SQL queries to `src/memory/db.ts` if schema change needed
- Increment version in `runMigrations()` for automatic migration
- Add methods to `MemoryManager` interface in `src/memory/memory-manager.ts`
- Implement in factory function in same file

**New LLM Provider:**
- Create file: `src/llm/your-provider.ts`
- Implement `LLMProvider` interface: `chat(messages, tools?, complexity?): Promise<LLMChatResult>`
- Return: `{message: ChatMessage, model: string}`
- Update `src/index.ts` to instantiate your provider instead of OpenRouterProvider

**Utilities/Helpers:**
- Shared helpers: `src/` root level (e.g., `src/logger.ts`)
- Domain-specific: Within layer folder (e.g., `src/llm/model-router.ts` for LLM logic)
- Avoid: Dedicated `utils/` or `helpers/` folders - keep flat unless > 5 files

## Special Directories

**data/:**
- Purpose: Runtime state (database, uploads)
- Generated: Yes (created by app at startup)
- Committed: No (gitignored, but may have `.gitkeep`)

**dist/:**
- Purpose: TypeScript compilation output
- Generated: Yes (by `npm run build`)
- Committed: No (gitignored)

**node_modules/:**
- Purpose: Installed dependencies
- Generated: Yes (by `npm install`)
- Committed: No (gitignored)

## Module Resolution

**Import Style:**
- ES module syntax only: `import` / `export`
- File extensions required: `import { runAgent } from "./agent.ts"`
- Package aliases: None configured (bare specifiers work for node_modules)
- Relative paths: `./relative/path.ts` for same-directory or parent navigation

**Example:**
```typescript
// ✓ Correct
import { config } from "./config.js";
import { runAgent } from "./agent/agent.js";
import { Tool } from "./tools/tool-types.js";

// ✗ Wrong
import { config } from "./config";           // missing extension
import config from "./config";              // wrong default
import { config } from "@/config";          // no path aliases
```
