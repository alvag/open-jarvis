**Module-level:**

- Export: `export function setApprovalGate(gate: ApprovalGate): void { approvalGateRef = gate; }`

**Tool definition:**
- name: `"execute_command"`
- description: `"Execute a shell command or local script on the user's Mac. Commands are classified as safe (runs immediately), risky (requires user approval), or blocked (never runs). Pipes, &&, and ; are NOT supported — use multiple tool calls for chaining. Scripts (.sh, .py, .ts) are always risky and require approval. Default working directory is the user's home directory."`
- parameters (type: "object"):
  - `command` (string, required): "The executable or script path to run (e.g. 'ls', 'git', '/Users/max/scripts/backup.sh')"
  - `args` (string): "Space-separated arguments for the command (e.g. '-la /tmp'). Pipes, &&, ; are not allowed."
  - `cwd` (string): "Working directory for the command. Defaults to user home directory."

**Execute function logic (in order):**

1. Parse inputs:
   - `command = args.command as string`
   - `rawArgs = (args.args as string | undefined) ?? ""`
   - `argList = rawArgs.trim() ? rawArgs.split(/\s+/) : []`
   - `cwd = (args.cwd as string | undefined) ?? homedir()`

2. Shell metacharacter check:
   - `const SHELL_META = /[|;&\`$(){}[\]<>]/;`
   - If `SHELL_META.test(rawArgs)` or `SHELL_META.test(command)`: return `{ success: false, data: null, error: "Shell metacharacters (|, ;, &, $, etc.) are not allowed. Use separate tool calls for chaining." }`

3. Script path validation:
   - If command ends with .sh, .py, or .ts AND `!isAbsolute(command)`: return error "Script paths must be absolute"

4. Classify command:
   - `const classification = classifyCommand(command, argList)`
   - Log: `log("info", "execute_command", "Command classified", { command, args: argList, classification })`

5. If `classification === "blocked"`:
   - `const reason = getBlockReason(command, argList) ?? "Command is in the blocked list"`
   - Return `{ success: false, data: null, error: \`Command blocked: ${reason}\` }`

6. If `classification === "risky"`:
   - If `!approvalGateRef`: return `{ success: false, data: null, error: "Approval gate not initialized. Cannot execute risky commands." }`
   - Determine reason string: if script -> "Script execution always requires approval"; else -> "Command classified as risky (modifies system state)"
   - `const approved = await approvalGateRef.request({ command, args: argList, cwd, reason, userId: context.userId, sendApproval: ... })`
   - The `sendApproval` callback is a function that the approval gate calls — it receives `(text: string, approvalId: string)`. For now, store a `sendApprovalFn` module-level setter (see below).
   - If `!approved`: return `{ success: false, data: null, error: "Command denied by user." }`

7. Execute with execFile:
   - For scripts: detect interpreter by extension:
     - `.sh` -> command = `"/bin/bash"`, prepend script path to argList
     - `.py` -> command = `"python3"`, prepend script path to argList
     - `.ts` -> command = `"tsx"`, prepend script path to argList
   - `const { stdout, stderr } = await execFileAsync(resolvedCommand, resolvedArgs, { shell: false, timeout: 30_000, cwd, maxBuffer: 10 * 1024 * 1024, env: process.env })`
   - Combine stdout + stderr: `const combined = [stdout, stderr].filter(Boolean).join("\n").trim()`
   - Truncation: if `combined.length > OUTPUT_LIMIT`, truncate with notice: `combined.slice(0, OUTPUT_LIMIT) + \`\n\n[Output truncated at ${OUTPUT_LIMIT} chars. ${combined.length - OUTPUT_LIMIT} chars omitted.]\``
   - Return `{ success: true, data: { output, truncated: combined.length > OUTPUT_LIMIT, exit_code: 0 } }`

8. Error handling (catch block):
   - If `error.killed` or `error.code === "ETIMEDOUT"`: return timeout error
   - Otherwise: extract stdout/stderr from error, return with exit code

**Additional module-level setter for sendApproval:**
- `let sendApprovalFn: ((userId: string, text: string, approvalId: string) => Promise<void>) | null = null;`
- `export function setSendApproval(fn: (userId: string, text: string, approvalId: string) => Promise<void>): void { sendApprovalFn = fn; }`

5. In `start()` method, BEFORE `this.bot.start();` and AFTER the existing command handlers, add the callback_query handler:
   ```typescript
   // Approval gate callback handler (registered once at startup)
   this.bot.on("callback_query:data", async (ctx) => {
     const data = ctx.callbackQuery.data;
     const userId = ctx.from!.id;

     if (!this.allowedUserIds.has(userId)) {
       await ctx.answerCallbackQuery({ text: "Access denied." });
       return;
     }

     const [action, id] = data.split(":", 2);
     if ((action === "approve" || action === "deny") && id && this.approvalGate) {
       const approved = action === "approve";
       this.approvalGate.handleCallback(id, approved);
       await ctx.answerCallbackQuery({
         text: approved ? "Comando aprobado." : "Comando denegado.",
       });
       // Remove inline buttons from the original message
       await ctx.editMessageReplyMarkup({ reply_markup: undefined }).catch(() => {});
     }
   });
   ```

   IMPORTANT: This handler must be registered ONCE in `start()`, NOT inside `handleIncoming()`. See Pitfall 4 in RESEARCH.md.
