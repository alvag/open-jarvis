**loadToolManifest function logic:**
1. Resolve path: `manifestPath ?? process.env.MANIFEST_PATH ?? "./tool_manifest.json"` then `path.resolve()`
2. If file does not exist (`!existsSync`): `log("info", "manifest-loader", "tool_manifest.json not found — starting without manifest tools", { path: resolved })` and return
3. Try `JSON.parse(readFileSync(resolved, "utf-8"))`. On catch: `log("error", "manifest-loader", "Failed to parse tool_manifest.json — starting without manifest tools", { path, error: err.message })` and return
4. Validate parsed value is an array. If not: log error "tool_manifest.json must be a JSON array" and return
5. Loop entries: skip if `entry.enabled === false`
6. Validate required fields exist: `name` (string), `description` (string), `parameters` (object), `handler_path` (string). If missing: `log("error", "manifest-loader", "Invalid manifest entry — missing required fields, skipping", { entry: entry.name ?? "unknown" })` and continue
7. Resolve handler_path with `path.resolve(entry.handler_path)`. If `!existsSync(handlerPath)`: `log("error", "manifest-loader", "Handler not found for tool \"${entry.name}\" — skipping", { handlerPath })` and continue
8. Build a `Tool` object via `buildManifestTool(entry, handlerPath)` (see below)
9. Try `registry.register(tool)`. On catch: `log("error", "manifest-loader", "Tool \"${entry.name}\" collides with existing tool — skipping (built-in has priority)")` and continue
10. On success: `log("info", "manifest-loader", "Registered manifest tool: ${entry.name}")`

**buildManifestTool function** (private, same file):
- Determine interpreter by extension: `.py` -> `"python3"`, `.sh` -> `"/bin/bash"`, `.ts` -> `"tsx"`, else run directly as executable
- Return a `Tool` object with:
  - `definition`: `{ name: entry.name, description: entry.description, parameters: entry.parameters as JsonSchema }`
  - `execute(args, context)`: async function that:
    1. Imports `classifyCommand` and `getBlockReason` from `"../../security/command-classifier.js"` (dynamic import to avoid circular deps -- actually, use static import at top of file since there's no circular dependency)
    2. Calls `classifyCommand(handlerPath, [])` to classify the handler path
    3. If `"blocked"`: return `{ success: false, data: null, error: "Handler path is blocked by security policy" }`
    4. If `"risky"`: Import `approvalGateRef` pattern -- BUT since `execute-command.ts` owns the gate refs and they are module-private, manifest tools must use a shared mechanism. Solution: import `setApprovalGate` is already called in index.ts. For manifest tools, the simplest approach is to import the approval gate ref via a shared module. HOWEVER, looking at execute-command.ts, the gate/send functions are module-private. The cleanest solution per research: manifest script execution should delegate to the SAME approval+execution flow. Create a helper that reuses the pattern:
       - Store a module-level `approvalGateRef` and `sendApprovalFn` and `sendResultFn` in manifest-loader.ts, with exported setters `setManifestApprovalGate`, `setManifestSendApproval`, `setManifestSendResult` (same pattern as execute-command.ts)
       - For "risky" classification: use the same fire-and-forget approval pattern from execute-command.ts lines 140-176
       - For "safe" classification: execute directly (scripts are always risky by classifyCommand since they end in .sh/.py/.ts, so this path may not trigger, but include for completeness)
    5. Spawn child process: `spawn(interpreter, [handlerPath], { stdio: ["pipe", "pipe", "pipe"], env: process.env, timeout: 30_000 })`
    6. Write `JSON.stringify(args)` to `child.stdin`, then `child.stdin.end()`
    7. Collect stdout and stderr
    8. On `close` with code !== 0: return `{ success: false, data: null, error: "Handler exited with code ${code}: ${stderr}" }`
    9. On `close` with code === 0: try `JSON.parse(stdout)` as ToolResult. On parse failure: return `{ success: false, data: null, error: "Handler stdout is not valid JSON: ${stdout.slice(0, 200)}" }`
    10. On spawn `error`: return `{ success: false, data: null, error: "Failed to spawn handler: ${err.message}" }`

**substituteEnvVars function** (private):
```typescript
function substituteEnvVars(value: string, serverName: string): string | null {
  let ok = true;
  const result = value.replace(VAR_PATTERN, (_, varName: string) => {
    const val = process.env[varName];
    if (val === undefined) {
      log("error", "mcp-config-loader", `Undefined env var "\${${varName}}" in server "${serverName}" — skipping server`);
      ok = false;
      return "";
    }
    return val;
  });
  return ok ? result : null;
}
```
This handles partial substitution: `"Bearer ${TOKEN}"` becomes `"Bearer actual-value"`. If ANY variable is undefined, the entire server is skipped (returns null).

2. If file does not exist: `log("info", "mcp-config-loader", "mcp_config.json not found — no MCP servers configured", { path: resolved })` and return `[]`

4. Validate parsed value has `mcpServers` key as object. If not or missing: `log("info", "mcp-config-loader", "No mcpServers key found in mcp_config.json")` and return `[]`
5. Loop `Object.entries(raw.mcpServers)`:
   a. If `entry.enabled === false`: `log("info", "mcp-config-loader", "MCP server \"${name}\" is disabled — skipping")` and continue
   b. Determine type: `(entry.type as string) ?? "stdio"` -- default "stdio" per user decision (compatible with Claude Desktop format)
   c. Validate type is either "stdio" or "streamable-http". If not: `log("error", "mcp-config-loader", "Unknown server type \"${type}\" for \"${name}\" — skipping")` and continue
   d. For stdio: validate `command` field exists (string). If missing: `log("error", "mcp-config-loader", "stdio server \"${name}\" missing command field — skipping")` and continue
   e. For streamable-http: validate `url` field exists (string). If missing: `log("error", "mcp-config-loader", "streamable-http server \"${name}\" missing url field — skipping")` and continue
   f. Substitute `${VAR}` in `env` fields: iterate `Object.entries(entry.env)`, call `substituteEnvVars(v, name)` for each value. If any returns null, set `failed = true` and break. If failed, continue to next server.
   g. Substitute `${VAR}` in `headers` fields (same logic as env). Only applies to streamable-http servers.
   h. Push `McpServerConfig` to result array:

      ```typescript
      servers.push({
        name,

        command: entry.command,
        args: entry.args,
        url: entry.url,
        headers: resolvedHeaders,
        env: resolvedEnv,
      });
      ```
   i. `log("info", "mcp-config-loader", "Loaded MCP server config: ${name}", { type: resolvedType })`
6. Return `servers`
