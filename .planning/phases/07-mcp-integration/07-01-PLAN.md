- Accept `McpServerConfig[]` in constructor, store as private `configs`
- Maintain private `clients: McpClient[]` array (only connected clients)
- Export `McpStartupSummary` interface: `{ connected: number; failed: number; toolsRegistered: number; errors: string[] }`
- Export `McpManager` class

**`connectAll(registry: ToolRegistry): Promise<McpStartupSummary>`**
- If `configs.length === 0`, return `{ connected: 0, failed: 0, toolsRegistered: 0, errors: [] }` immediately (no logging — index.ts handles the consolidated log)
- Map each config to `connectOne(cfg, registry)` and run all via `Promise.allSettled`
- Aggregate results into McpStartupSummary: fulfilled = connected + toolsRegistered sum; rejected = failed + error message pushed to errors array
- Return the summary

**`connectOne(cfg: McpServerConfig, registry: ToolRegistry): Promise<number>`** (private)
- Create `new McpClient(cfg)`
- Apply 10-second connection timeout using `Promise.race` with `client.connect()` and a `setTimeout` reject
- Call `client.listTools()` to get tools
- Call `adaptMcpTools(tools, client, cfg.name)` to get adapted Tool objects
- Register each adapted tool in `registry`, catching throws (ToolRegistry throws on collision) and logging warnings with category "mcp"
- Push client to `this.clients`
- Log connection success: `log("info", "mcp", \`Connected: ${cfg.name}\`, { toolsRegistered: registered, totalExposed: tools.length })`
- Return the count of registered tools

**`disconnectAll(): Promise<void>`**
- `await Promise.allSettled(this.clients.map((c) => c.disconnect()))`
- `log("info", "shutdown", "MCP connections closed")`
